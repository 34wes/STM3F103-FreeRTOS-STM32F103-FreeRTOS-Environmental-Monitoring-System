STM32F103-FreeRTOS 环境监测系统（光照+人体红外+WiFi TCP上报）
📌 项目简介
本项目基于 STM32F103C8T6 主控、HAL 库开发，搭载 FreeRTOS 实时操作系统，实现环境光照采集、人体红外感应、TCP 网络数据上报多功能嵌入式监测系统。硬件对接 BH1750 数字光照传感器、HC-SR501 人体红外热释电传感器、ESP8266 WiFi 模组，通过多任务分时调度完成传感器数据采集、阈值异常报警、云端 TCP 数据定时上传，适用于智能照明、环境安防、室内状态监测等场景。
项目针对 ESP8266 频繁断连、AT 指令丢包问题做了专属容错优化，搭建可靠的 WiFi 重连与指令重传校验机制，保障 TCP 长连接稳定性，具备极强的工程实用性与抗干扰能力。
🛠 开发环境与技术栈
硬件平台
- 主控芯片：STM32F103C8T6（ARM Cortex-M3）
- 光照传感器：BH1750（I2C 数字光照采集，高精度）
- 人体感应模块：HC-SR501 热释电红外传感器
- 通信模组：ESP8266 WiFi 模组（AT 指令驱动）
软件工具与技术
- 底层驱动：STM32 HAL 库标准驱动开发
- 实时系统：FreeRTOS 多任务调度
- 开发编辑：VS Code（代码编写、语法检查）
- 编译调试：Keil MDK5
- 版本管理：Git / GitHub 版本管控
- 通信协议：TCP 长连接、ESP8266 AT 指令协议
🎯 核心功能架构（FreeRTOS 三任务分区）
项目采用 FreeRTOS 模块化多任务设计，任务解耦、独立调度，互不阻塞，保证系统实时性与稳定性。
1. 光照采集任务
- 基于 I2C 驱动 BH1750，定时采集环境光照强度数据（lux）
- 数据滤波处理，消除环境瞬时干扰，保证采集精度
- 实时更新全局环境变量，为报警、上传任务提供数据支撑
2. 阈值报警任务
- 结合光照阈值 + 人体红外感应状态双条件判断
- 自定义光照上下限阈值，实现环境异常状态判定
- 支持人体移动检测，联动环境状态，实现智能监测逻辑
- 异常状态标记，同步上传至 TCP 服务器
3. TCP 数据上报任务
- ESP8266 模组配网，建立与远端服务器的 TCP 长连接
- 定时打包光照数据、人体状态、设备状态并上报
- 支持服务器下行指令响应，可拓展远程控制功能
🔥 项目核心优化亮点（解决工程痛点）
1. ESP8266 断连自愈机制
针对嵌入式场景常见的 ESP8266 频繁掉线、网络抖动、AT 指令超时问题，定制容错机制：
- AT 指令超时重传、应答校验机制，避免指令丢包失效
- 实时检测 TCP 连接状态，断连自动触发重连逻辑
- 配网失败、服务器掉线、模组异常分层处理，杜绝卡死死机
- 最终实现 TCP 双向高稳定长连接，长时间运行无断连、无死机
2. FreeRTOS 稳定多任务设计
- 任务优先级合理分层，采集任务、报警任务、上传任务互不抢占冲突
- 采用任务延时阻塞式调度，降低 CPU 占用率，功耗更低
- 全局变量加临界区保护，避免多任务数据竞争冲突
3. 标准化工程开发流程
- VS Code 高效编写 + Keil MDK 精准编译调试
- Git 分支管理、版本迭代、代码回溯，规范开发流程
- 驱动分层封装：硬件层、任务层、通信层解耦，可读性、可移植性极强

⚙️ 运行流程
1. 系统上电，初始化时钟、串口、I2C、GPIO、外设
2. ESP8266 初始化，AT 指令校验，连接 WiFi 热点
3. 主动连接远端 TCP 服务器，建立长连接
4. FreeRTOS 启动三大业务任务，并行运行
5. 定时采集光照、人体红外数据，阈值判断异常报警
6. 周期性打包环境数据，通过 TCP 上传至服务器
7. 网络异常自动重连，保障系统 7*24h 稳定运行
